Experiment Task 1: Learning Rate Effects

Let's start by focusing on the Learning Rate. This is arguably the most critical hyperparameter in Gradient Descent.

Task: Keep the 'Maximum Iterations' and 'Convergence Threshold' at their default settings. Vary the Learning Rate slider slowly from a very small value (close to 0) to a large value (close to 1.0 or even higher). Observe what happens to Gradient Descent's behavior as you change the learning rate.

  1. Small Learning Rate: Set the Learning Rate to a very small value (e.g., 0.001 or lower). Run Gradient Descent. What do you observe about the speed of convergence? How many iterations does it take?
  2. Medium Learning Rate: Gradually increase the Learning Rate. Find a 'medium' value that seems to make Gradient Descent converge reasonably quickly and smoothly. Observe the path and convergence speed.
  3. Large Learning Rate: Keep increasing the Learning Rate to larger values (e.g., 0.5, 0.8, 1.0 or higher). What happens now? Do you see oscillations? Does Gradient Descent still converge, or does it diverge (cost function increase)?
  4. Divergence Point: Try to find a Learning Rate value that is so large that Gradient Descent clearly diverges, meaning the cost function starts increasing instead of decreasing.

Stop and Think

After experimenting with different learning rates, reflect on your observations. What general conclusions can you draw about the effect of the learning rate on Gradient Descent?

Key observations should include: Small learning rates lead to slow convergence, but stable descent. Large learning rates can lead to faster initial progress but oscillations, overshooting, and potential divergence. Finding a 'good' learning rate is about balancing speed and stability.

Experiment Task 2: Maximum Iterations

Now, let's explore the Maximum Number of Iterations (N). This hyperparameter sets a limit on how long Gradient Descent will run.

Task: Set the 'Learning Rate' to a value that you found to work reasonably well in Task 1 (a 'medium' learning rate). Now, significantly reduce the 'Maximum Iterations' value (e.g., to 10, 20, 50 iterations). Observe what happens when you limit the number of iterations.

  1. Limited Iterations: With a small 'Maximum Iterations' value, run Gradient Descent. Does it reach convergence within the limited iterations? Does it reach the minimum of the cost function? How does the final cost function value compare to when you used a larger number of iterations?
  2. Insufficient Iterations: Try different small values for 'Maximum Iterations'. Can you find a value so small that Gradient Descent stops very early, far from the minimum, and with a relatively high cost function value?

Stop and Think

Reflect on the role of Maximum Iterations. Why is it important to have this hyperparameter? What are the trade-offs of setting it too low vs. relying solely on the Convergence Threshold?

Key insights should include: Maximum Iterations prevents infinite loops and limits training time. Setting it too low might stop training prematurely before convergence. It's a safeguard when convergence is slow or uncertain, or when a time limit is needed.

Experiment Task 3: Convergence Threshold

Finally, let's investigate the Convergence Threshold (\(\epsilon\)). This hyperparameter determines when Gradient Descent considers itself 'converged'.

Task: Set a 'good' Learning Rate and a reasonably large 'Maximum Iterations' value (so iterations are not the limiting factor). Now, vary the 'Convergence Threshold' slider from a very small value (e.g., 0.0001) to a larger value (e.g., 0.1). Observe how the threshold affects convergence.

  1. Small Threshold: Set a very small Convergence Threshold. Run Gradient Descent. How many iterations does it take to converge? How small does the gradient magnitude become at convergence? How precise is the final solution (cost function value)?
  2. Large Threshold: Increase the Convergence Threshold to a larger value. Run Gradient Descent again. How does the number of iterations change? How does the final cost function value compare to the small threshold case? Is convergence reached 'earlier' or 'later'?
  3. Trade-off: What is the trade-off between setting a very small and a larger Convergence Threshold?

Stop and Think

Reflect on the role of the Convergence Threshold. How does it influence the 'precision' of the solution and the training time? What are the trade-offs of setting it too small vs. too large?

Key takeaways should include: Smaller thresholds lead to more precise solutions (lower cost) but might require more iterations (longer training time). Larger thresholds lead to faster convergence (fewer iterations) but potentially less precise solutions. Choosing the threshold involves balancing precision and training efficiency.
